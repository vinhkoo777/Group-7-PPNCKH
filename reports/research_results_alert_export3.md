# Beyond Classification: Research Results on `alert_export3`

## Experimental Setup

The framework was trained and evaluated using the new Wazuh exported alert dataset:

- Wazuh dataset: `data/raw/alert_export3.json`
- Wazuh alert records: `358,299`
- Hybrid MachineLearningCVE/CIC-IDS2017 records added: `66,385`
- Total processed alerts: `424,684`
- Execution command:

```powershell
$env:RAW_ALERTS_PATH='data/raw/alert_export3.json'
$env:HYBRID_MLCVE_ENABLED='1'
python src\main.py
```

The experiment follows an offline/batch design. The system does not perform real-time Wazuh re-ingestion, Wazuh API integration, custom Wazuh rule deployment, or Wazuh dashboard integration.

## Model Performance

The Random Forest classifier achieved strong overall classification performance on the test split:

- Accuracy: `0.99`
- Weighted precision: `0.99`
- Weighted recall: `0.99`
- Weighted F1-score: `0.99`
- Macro F1-score: `0.90`

Per-class results show that large classes such as `Benign`, `WebAttack`, `DoS`, `DDoS`, `BruteForce`, `PortScan`, and `Bot` were classified very well. Smaller classes such as `Recon` and `SuspiciousProcess` showed weaker performance due to limited support and class imbalance.

## Research Interpretation

The result supports the "Beyond Classification" argument. The system does not stop at predicting an attack label; it converts model confidence, anomaly evidence, Wazuh rule severity, recurrence, and asset criticality into a composite PriorityScore. This score is then used to route alerts into SOC-oriented queues.

The experiment demonstrates three main findings:

1. Random Forest can classify rule-labeled and pseudo-Wazuh hybrid alerts with high weighted performance.
2. Isolation Forest adds a separate anomaly-evidence signal, identifying 20,035 anomalous alerts that should remain visible for analyst review.
3. Composite routing produces a highly precise high-priority queue, but the current thresholds are conservative and route most alerts into anomaly_review.

## Limitations

- Labels are generated by rule-based mapping and pseudo-Wazuh conversion, not by verified SOC analyst ground truth.
- The hybrid MachineLearningCVE records are transformed into alert-like Wazuh records, so they do not perfectly represent native Wazuh alert semantics.
- The current threshold configuration produces low alert volume reduction because most alerts remain in `anomaly_review`.
- Feedback is simulated through `feedback.csv`; it is not yet used for automatic retraining.
- The system remains an offline/batch research prototype, not a production Wazuh integration.

## Suggested Next Step

To improve SOC operational value, the next experiment should tune routing thresholds and weights. The key objective is to increase `low_priority` routing for clear benign/low-risk alerts while preserving high precision in `high_priority` and keeping suspicious rare events in `anomaly_review`.
